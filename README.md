# Health Signals

Интерактивный одностраничный дашборд по личным данным **Apple Health за 2017–2026**.
Визуализирует тренды сердца, активности, сна, веса, восстановления и шумовой нагрузки на основе экспорта из приложения «Здоровье».

🔗 **Live:** https://tomilov-sergei.github.io/health-dashboard/

> ⚠️ Это не медицинское заключение. Данные собраны с потребительских устройств (Apple Watch, iPhone, Oura) и могут содержать погрешности и артефакты разных источников. Любые тревожные изменения стоит обсудить с врачом.

---

## О проекте

Архив Apple Health — это `export.xml` на ~2,6 ГБ с более чем **10 млн записей** и **88 типами метрик**. Сырые данные невозможно осмыслить вручную, поэтому проект:

1. Потоково разбирает гигабайтный XML и считает годовые/месячные агрегаты.
2. Очищает данные от артефактов (например, задвоение сна от двух устройств).
3. Показывает главные тренды и формулирует инсайты в виде наглядного дашборда.

### Что внутри (9 разделов)

| № | Раздел | Что показывает |
|---|--------|----------------|
| 01 | Сердце и восстановление | Пульс покоя, VO₂Max, HRV, дыхание + SpO₂ |
| 02 | Активность | Шаги, активные калории, минуты упражнений, дистанция |
| 03 | Тренировки | По годам, топ видов, помесячно за 2 года (1206 сессий) |
| 04 | Вес | Реальная динамика 2017–2026 (циклы 66–75 кг) |
| 05 | Сон | По годам и за последние 12 месяцев (пересчитанные интервалы) |
| 06 | Шумовая нагрузка | Наушники vs окружающий шум, порог риска 80 дБ |
| 07 | ЭКГ | 17 записей с классификацией ритма |
| 08 | Главные инсайты | Цветовая маркировка: отлично / внимание / контекст |
| 09 | Рекомендации | Что стоит сделать |

## Ключевые выводы

- 🟢 **Активность на пике** — шаги выросли с ~8 100/день (2017) до ~20 600/день (2026).
- 🟢 **Сон стабилен** — 8,0 ч/ночь в 2026 против 6,8 ч в 2021.
- 🟡 **Пульс покоя ползёт вверх** — с ~56 (2018) до 60,4 уд/мин (2026).
- 🟡 **VO₂Max просел** — с 41,3 до 34,0 мл/мин·кг (−18%).
- 🔵 **Вес цикличен** — реальные периоды набора и сброса, сейчас 72,7 кг.
- 🟢 **ЭКГ в норме** — преобладает синусовый ритм.

Связка «растущий пульс покоя + падающий VO₂Max + стабильный HRV» — сигнатура того, что нагрузка обгоняет восстановление.

## Технологии

- **Чистый HTML / CSS / JS** — без сборщиков и фреймворков, полностью статический сайт.
- **[Chart.js 4](https://www.chartjs.org/)** (через CDN) — линейные и столбчатые графики, двойные оси, градиентные заливки.
- **Шрифты:** [Manrope](https://fonts.google.com/specimen/Manrope) (текст) + [Spline Sans Mono](https://fonts.google.com/specimen/Spline+Sans+Mono) (числа и подписи) через Google Fonts.
- **Хостинг:** GitHub Pages (ветка `master`, корень репозитория).
- **Дизайн:** тёмная тема, акцент teal `#34d6a8`, табличные цифры (`tabular-nums`), адаптивная вёрстка (mobile-first), цветовая маркировка инсайтов.

Данные обработаны на Python (потоковый `xml.etree.ElementTree.iterparse`) и встроены прямо в страницу как JS-объект — никаких внешних запросов и проблем с CORS.

### Формат данных (`data_embed.js`)

Глобальный объект `window.HEALTH_DATA` с ключами:

- Годовые средние: `restingHR`, `vo2max`, `hrv`, `heartRate`, `respRate`, `spo2`, `envNoise`, `headphoneNoise`
- Годовые суточные суммы: `steps`, `distance`, `activeEnergy`, `exerciseMin`
- `sleep` (по годам), `sleepRecent` (12 месяцев) — часы сна за ночь
- `weight` — массив `[месяц, кг]` ключевых замеров
- `workoutsByYear`, `workoutsByMonth`, `workoutTypes`, `totalWorkouts`
- `ecg` — массив `[дата, классификация]`
- `totalRecords`, `recordsSum`, `typesCount` — общая статистика архива

## Методология и оговорки

- **Сон** пересчитан через объединение перекрывающихся интервалов (merge intervals), чтобы убрать задвоение, когда сон одновременно пишут Apple Watch и Oura. Иначе суммы были завышены в 1,5–3 раза.
- **Вес** — редкие замеры из разных источников (iPhone, Health, Oura, сторонние приложения). Одна явная опечатка (56 → 65 кг, окт 2024) исправлена вручную. Динамика реальная, но точек мало — для чёткой кривой нужно взвешиваться регулярно.
- **VO₂Max / пульс / HRV** — оценки Apple Watch, зависят от типа активности и условий замера.
- Личные данные (имя, дата рождения) намеренно **не выводятся** на сайт — только агрегированные тренды.

## Как обновить

1. Заново экспортировать данные из приложения «Здоровье» (Профиль → Экспортировать медданные).
2. Прогнать парсинг `export.xml` и обновить `data_embed.js` (формат `window.HEALTH_DATA={...};`).
3. Локальный просмотр: `python3 -m http.server 8000`, открыть `http://localhost:8000`.
4. Закоммитить и запушить — GitHub Pages пересоберёт сайт автоматически за 1–2 минуты.
